---
title: 'Get Client'
api: 'GET /api/clients/:id'
description: 'Retrieve detailed information about a specific client'
---

## Request

### Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the client to retrieve. Must be a valid integer.
  
  **Example:** `1`
</ParamField>

## Response

<ResponseField name="id" type="number">
  Client's unique identifier
</ResponseField>

<ResponseField name="name" type="string">
  Client's business name or name
</ResponseField>

<ResponseField name="status" type="number">
  Client status. `1` for active, `0` for inactive
</ResponseField>

## Error Responses

### 400 Bad Request

Returned when the `id` parameter is not a valid integer.

```json
{
  "statusCode": 400,
  "message": "Validation failed (numeric string is expected)",
  "error": "Bad Request"
}
```

### 404 Not Found

Returned when a client with the specified ID does not exist.

```json
{
  "statusCode": 404,
  "message": "Cliente con ID {id} no encontrado",
  "error": "Not Found"
}
```

## Example Request

```bash cURL
curl -X GET https://api.example.com/api/clients/1
```

## Example Response

```json 200 OK
{
  "id": 1,
  "name": "Empresa SAC",
  "status": 1
}
```

```json 404 Not Found
{
  "statusCode": 404,
  "message": "Cliente con ID 999 no encontrado",

```
