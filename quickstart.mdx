# Quickstart Guide

Get the API up and running on your local machine in just a few steps.

<Note>
  This guide uses **Bun** as the JavaScript runtime. Make sure you have Bun installed before proceeding. See the [Installation](/installation) page for prerequisites.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/JonathanLara27/backend-sinapsis-test
    cd backend-sinapsis-test
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using Bun:

    ```bash
    bun install
    ```

    This will install:
    - NestJS core and platform packages
    - TypeORM and MySQL driver
    - Validation libraries (class-validator, class-transformer)
    - Swagger documentation tools
    - Serverless framework and adapters
  </Step>

  <Step title="Start the database">
    Launch a MySQL 8.0 container using Docker Compose:

    ```bash
    docker-compose up -d
    ```

    This creates a MySQL instance with:
    - Database: `sinapsis_db`
    - Root password: `root`
    - Port: `3307` (to avoid conflicts with local MySQL)

    <Info>
      The database runs on port **3307** instead of the default 3306 to prevent conflicts with existing MySQL installations.
    </Info>
  </Step>

  <Step title="Start the development server">
    Run the API in development mode with hot reload:

    ```bash
    bun run start:dev
    ```

    The server will start with:
    - API endpoints at `http://localhost:3000/api`
    - Swagger documentation at `http://localhost:3000/api/docs`
    - Auto-reload on file changes

    <Tip>
      You can also run in serverless mode with `bun run start:sls` to simulate AWS Lambda locally.
    </Tip>
  </Step>
</Steps>

## Make Your First API Call

Once the server is running, you can create your first marketing campaign.

### Create a Campaign

Send a POST request to create a campaign with multiple messages:

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/api/campaigns \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Campaña Navidad 2026",
    "scheduledAt": "2026-12-24T10:00:00Z",
    "userId": 1,
    "messages": [
      "Oferta Exclusiva - 50% descuento en todos los productos",
      "Envío gratis en compras superiores a $100"
    ]
  }'
```

```javascript JavaScript (Fetch)
fetch('http://localhost:3000/api/campaigns', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Campaña Navidad 2026',
    scheduledAt: '2026-12-24T10:00:00Z',
    userId: 1,
    messages: [
      'Oferta Exclusiva - 50% descuento en todos los productos',
      'Envío gratis en compras superiores a $100'
    ]
  })
})
.then(res => res.json())
.then(data => console.log(data));
```

```python Python (requests)
import requests

response = requests.post(
    'http://localhost:3000/api/campaigns',
    json={
        'name': 'Campaña Navidad 2026',
        'scheduledAt': '2026-12-24T10:00:00Z',
        'userId': 1,
        'messages': [
            'Oferta Exclusiva - 50% descuento en todos los productos',
            'Envío gratis en compras superiores a $100'
        ]
    }
)

print(response.json())
```
</CodeGroup>

### Request Body Fields

The `CreateCampaignDto` requires the following fields:

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| `name` | string | Campaign name | Required, non-empty |
| `scheduledAt` | string | Schedule date/time | Required, ISO 8601 format |
| `userId` | number | ID of user creating the campaign | Required, integer |
| `messages` | string[] | Array of campaign messages | At least 1 message, each min 5 chars |

<Warning>
  Messages are automatically trimmed and validated. Empty messages or messages shorter than 5 characters will be rejected with a `400 Bad Request` error.
</Warning>

### Expected Response

On success, you'll receive a `201 Created` response with the campaign details:

```json
{
  "id": 1,
  "name": "Campaña Navidad 2026",
  "scheduledAt": "2026-12-24T10:00:00.000Z",
  "userId": 1,
  "createdAt": "2026-03-05T12:00:00.000Z",
  "messages": [
    {
      "id": 1,
      "content": "Oferta Exclusiva - 50% descuento en todos los productos",
      "status": "pending"
    },
    {
      "id": 2,
      "content": "Envío gratis en compras superiores a $100",
      "status": "pending"
    }
  ]
}
```

## Explore the API Documentation

Visit the interactive Swagger UI to explore all available endpoints:

```
http://localhost:3000/api/docs
```

The Swagger UI provides:
- Complete API reference for all endpoints
- Request/response schemas
- Interactive "Try it out" functionality
- Example payloads for each endpoint

<Info>
  The Swagger documentation is automatically generated from the TypeScript code using decorators like `@ApiOperation`, `@ApiResponse`, and `@ApiProperty`.
</Info>

## Get Dashboard Reports

Retrieve message statistics grouped by delivery status:

```bash
curl "http://localhost:3000/api/reports/dashboard?month=2026-03"
```

You can optionally filter by client ID:

```bash
curl "http://localhost:3000/api/reports/dashboard?month=2026-03&clientId=1"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection errors">
    Make sure Docker is running and the MySQL container is up:
    
    ```bash
    docker ps
    ```
    
    You should see a `mysql:8.0` container running on port 3307.
  </Accordion>

  <Accordion title="Port already in use">
    If port 3000 is already in use, you can change it by setting the `PORT` environment variable:
    
    ```bash
    PORT=4000 bun run start:dev
    ```
  </Accordion>

  <Accordion title="Validation errors (400 Bad Request)">
    The API uses strict validation. Common issues:
    - Messages array is empty
    - Individual messages are shorter than 5 characters
    - `scheduledAt` is not in ISO 8601 format
    - Missing required fields (`name`, `userId`, etc.)
  </Accordion>
</AccordionGroup>
